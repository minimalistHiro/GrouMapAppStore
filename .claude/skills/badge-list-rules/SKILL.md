# Badge List Rules

## 概要

バッジに関連するワード（「バッジ」「badge」「バッジ追加」「バッジ削除」「バッジ変更」「バッジ一覧」「バッジ定義」「レア度」など）が会話に出てきた場合、必ず `BADGE_LIST.md` を参照し、変更があれば追記・修正する。

バッジ定義の正となるソースは `lib/data/badge_definitions.dart` であり、`BADGE_LIST.md` はそのドキュメントとして常に同期を保つ。

## 手順

1. 依頼内容が「バッジ」「badge」「バッジ追加」「バッジ削除」「バッジ変更」「バッジ一覧」「バッジ定義」「レア度変更」「獲得条件」「バッジ画像」「バッジトリガー」に該当するか判断する。
2. 該当する場合は `/Users/kanekohiroki/Desktop/groumapapp/BADGE_LIST.md` を開いて現在のバッジ一覧を確認する。
3. コード変更（`lib/data/badge_definitions.dart` やトリガー実装箇所）を行った場合、以下の項目について `BADGE_LIST.md` を追記・修正する：
   - **バッジの追加**: 該当カテゴリのテーブルに新しいバッジ行を追加し、バッジ数サマリーを更新する。
   - **バッジの削除**: 該当カテゴリのテーブルからバッジ行を削除し、バッジ数サマリーを更新する。廃止扱いの場合は「（廃止・獲得不可）」の注記を付与する。
   - **バッジ名・獲得条件・レア度の変更**: 該当行を修正する。
   - **新カテゴリの追加**: 新しいセクション・テーブルを作成し、バッジ数サマリーに行を追加する。
   - **トリガー実装の追加・変更**: 「獲得処理」セクションのトリガー実装状況リストを更新する。
   - **画像関連の変更**: 「画像」セクションの生成済み数を更新する。
4. バッジ数サマリー（合計数）が実際のバッジ定義数と一致していることを確認する。
5. `badge_definitions.dart` の内容と `BADGE_LIST.md` の内容に差分がないことを確認する。

## 判定ルール

- バッジに関するワードが出てきた場合は、まず `BADGE_LIST.md` を読み込んで現状を把握する。
- コード変更を伴わない質問・確認のみの場合は、`BADGE_LIST.md` の内容を正として回答する。
- コード変更を伴う場合は、実装後に必ず `BADGE_LIST.md` を更新する。
- 変更が不要な場合は「BADGE_LIST.md の変更不要」とその理由を報告する。

- **バッジ獲得ロジック**: `lib/providers/badge_provider.dart`
- **バッジ画像保存先**: `assets/images/badges/{badgeId}.png`
