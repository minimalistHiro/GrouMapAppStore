# Agent Team Roadmap Update

## 概要

フェーズの進捗に合わせて `AGENT_TEAM_GUIDE.md` と `IMPLEMENTATION_ROADMAP.md` を最新状態に整理・更新する。
さらに、フェーズ完了に伴う変更内容を踏まえて関連マークダウンドキュメントも同時に整理する。

## 手順

ユーザーから「手順書を更新して」「フェーズが完了したのでガイドを整理して」「ロードマップを更新して」などの依頼があったら、以下の手順を順に実行する。

### Step 1: 現在の状態を把握する

以下のファイルを順に読み込み、現在の完了状態を把握する：

1. `/Users/kanekohiroki/Desktop/groumapapp/IMPLEMENTATION_ROADMAP.md` を読み込む
   - フェーズ0〜3 の各チェックボックスを確認する
   - `[x]` になっているものが完了済みタスク
   - `[ ]` のままのものが未完了タスク

2. `/Users/kanekohiroki/Desktop/groumapapp/AGENT_TEAM_GUIDE.md` を読み込む
   - 各フェーズの `> ステータス:` 行の現在状態を確認する

### Step 2: IMPLEMENTATION_ROADMAP.md を整理する

以下の判定を行い、必要な場合のみ更新する：

- フェーズ0 の全チェックボックスが `[x]` → フェーズ0 の見出しに `（完了）` を追記
- フェーズ1 の全チェックボックスが `[x]` → フェーズ1 の見出しに `（完了）` を追記
- フェーズ2 の全チェックボックスが `[x]` → フェーズ2 の見出しに `（完了）` を追記
- 一部のみ完了の場合はチェックボックスの状態をそのままにし、未完了タスクを明示する

### Step 3: AGENT_TEAM_GUIDE.md を整理する

IMPLEMENTATION_ROADMAP.md の状態に基づいて以下を更新する：

**完了済みフェーズの扱い:**
- 対象フェーズの `> ステータス: 未完了` を `> ステータス: 完了 (YYYY-MM-DD)` に変更する
  （日付は今日の日付 `currentDate` を使用）
- プロンプトセクションの先頭に以下のブロックを追加する：
  ```
  > このフェーズは完了済みです。以下のプロンプトはアーカイブとして保持しています。
  ```

**次フェーズの扱い:**
- 次に着手すべきフェーズの `> ステータス:` を `> ステータス: 実行中` に変更する
- そのフェーズのプロンプトセクションの先頭に「**現在実行中のフェーズ**」と明記する

**部分完了の扱い:**
- 一部のステップのみ完了している場合は、完了済みステップのプロンプトタイトルの末尾に `（完了済み）` を追記する

### Step 4: 関連マークダウンドキュメントを整理する

Step 1〜3 で把握した完了タスクの内容に基づき、以下の各ドキュメントを確認し、変更が必要な場合のみ更新する。

1. `/Users/kanekohiroki/Desktop/groumapapp/FIRESTORE.md` を読み込み、今回完了したタスクでFirestore関連（コレクション/フィールドの追加・変更・削除、ルール変更など）の変更がある場合は追記・修正する。
2. 今回完了したタスクに画面の作成・変更がある場合、`/Users/kanekohiroki/Desktop/groumapapp/USER_APP_SCREENS.md` を修正する。
3. 今回完了したタスクにサービス内容（機能追加・変更・削除、仕様変更など）に関わる箇所がある場合、`/Users/kanekohiroki/Desktop/groumapapp/SERVICE_FEATURES.md` を追記・修正する。
4. 今回完了したタスクにビジネスモデル（収益モデル・料金プラン・KPI定義・方針変更など）に関わる箇所がある場合、`/Users/kanekohiroki/Desktop/groumapapp/BUSINESS_MODEL.md` を追記・修正する。
5. 今回完了したタスクにサービス内容・機能・料金・仕様に関わる箇所がある場合、`/Users/kanekohiroki/Desktop/groumapapp/STORE_SALES_PRESENTATION.md`（店舗向け営業プレゼン資料）を追記・修正する。SERVICE_FEATURES.md や BUSINESS_MODEL.md の変更内容と整合するように更新する。
6. 今回完了したタスクにサービス内容・機能・仕様に関わる箇所がある場合、`/Users/kanekohiroki/Desktop/groumapapp/STORE_FEATURE_GUIDE.md`（店舗向け手渡し機能説明資料）を追記・修正する。**注意: 料金情報は記載しない。料金情報は STORE_SALES_PRESENTATION.md にのみ記載する。**
7. 今回完了したタスクにサービス内容・機能・仕様に関わる箇所がある場合、`/Users/kanekohiroki/Desktop/groumapapp/STORE_FEATURE_GUIDE_SIMPLE.md`（店舗向け簡潔版ガイド）を追記・修正する。簡潔版のため要点のみ反映する。**注意: 料金情報は記載しない。**
8. `/Users/kanekohiroki/Desktop/groumapapp/TODO_SETTINGS.md`（設定画面TODOリスト）を確認し、以下を実行する：
   - 今回完了したタスクで完了したTODO項目がある場合、「未着手」セクションから削除し「完了済み」セクションに完了日付付きで移動する。
   - 今回完了したタスクで新たに必要になった設定画面のTODOがある場合、適切なセクションに新規TODO項目を追記する。
   - TODO項目の内容が実装と乖離している場合は、実装に合わせて修正する。

## 判定ルール

- ファイルの変更が必要かどうかを判定してから実行する
- 変更が不要な場合は「変更不要」とその理由を報告する
- 日付は `currentDate`（今日の日付）を使用する
- ユーザーが「特定のフェーズが完了した」と明示した場合は、そのフェーズを完了扱いとする
- チェックボックスの状態とユーザーの発言が矛盾する場合は、ユーザーの発言を優先し確認を取る
- Step 4 の各ドキュメントは、今回完了したタスクに関係する場合のみ更新する（無関係なドキュメントは変更しない）
