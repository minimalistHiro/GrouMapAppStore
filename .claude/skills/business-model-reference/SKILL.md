# Business Model Reference

## 概要

ビジネスモデルに関する相談・検討・見直しを行う際に、必要なドキュメントを事前に読み込み、現状のビジネスモデルとサービス機能を把握した上で議論・提案を行う。

## 手順

1. 以下の2ファイルを必ず読み込む:
   - `/Users/kanekohiroki/Desktop/groumapapp/BUSINESS_MODEL.md`（ビジネスモデル定義）
   - `/Users/kanekohiroki/Desktop/groumapapp/SERVICE_FEATURES.md`（サービス機能一覧）
2. 両ファイルの内容を把握した上で、ユーザーの相談・質問に回答する。
3. ビジネスモデルに変更・更新がある場合は、`BUSINESS_MODEL.md` を追記・修正する。
4. サービス機能に関わる変更がある場合は、`SERVICE_FEATURES.md` も追記・修正する。

## 運用ルール

- ビジネスモデルの変更は必ずユーザーの承認を得てからドキュメントに反映する。
- 現状の決定事項・方針と矛盾しない提案を心がける。矛盾する場合はその旨を明示する。
